设计模式

发表于 2019-05-10 | 分类于 课程学习

  • 适配器模式： 通过增加一个接口，将已存在的子类封装起来，用户面向抽象的接口编程, 使用接口使得客户与具体方法的实现隔离

  • 装饰器模式： decorate抽象类实现主类接口，具体的decorate类继承自decorate抽象类，通过delegation实现功能共享

  • 外观模式： 提供一个统一的接口来取代一系列小接口调用，相当于对复杂 系统做了一个封装，简化客户端使用
    策略模式： 客户端对于某项功能有多种实现方式，将这些实现方式归一到一个接口的若干个子类方便调用。

  • 模板模式： 共性的步骤放在抽象类中公共实现，差异化的步骤在各个子类中实现 使用继承和重写实现模板设计模式 可以使用abstract和final来控制公共的和具体的方法

  • 迭代器模式：顺序访问对象的元素而不暴露其底层表示 iterator 实现 hasNext,next,remove Iterable接口：实现该接口的集合对象是可迭代遍历的

  • Factory Method

  • Abstract Factory Pattern 多个工厂方法套在一起只调用最高层次的的哪个

  • 构造器模式 挺傻逼的玩意正常写代码的都会不自觉的用到
  • 桥接模式： 通过delegation+inheritance 建立两个具体类之间的关系
    可以使用永久性delegation这样在运行时为二者建立联系
  • 代理模式： 某个对象被希望被客户访问所以设置代理模式，可以有效降低运行代价
  • 组合模式： 在同类型的对象之间建立起 树型层次结构，一个上层对象可包含多个下层对象
  • 观察者模式： 在subject类里面防一个list每次对这个类做修改的时候更新这个list中记录的相关类信息 保证list中记录的相关类无时无刻都能直接get到subject类的状态
  • 访问者设计模式：对特定类型的object的特定操作(visit)，在运行时将 二者动态绑定到一起，该操作可以灵活更改，无需更改被visit的类,日常写代码都能用到意识不到而已，就是把想要实现的功能的一部分放到类的外面不在内部写这样既可以方便大家一起调用也可以方便后期扩展、更改
  • 命令模式 安全的访问一些操作 信息隐藏
  • 责任链设计模式类似将所以需要改变的东西放到一个链表中逐个更改
  • state设计模式 负责管理程序运行状态的转移
  • memento设计模式 记录每一步的状态 可以用list
  • 单例模式 构造器设置为private 内部生成一个单一实例 首次调用时构造以后直接返回
  • 轻量模式：降低大量objects带来的代价 将同一个objects指向相同的地方 实现的时候设计一个可共享子类 一个不可共享子类. 把这个玩意当作工厂可以在这里当作一个存储池（如使用map）然后把公共的部分存到这里，取得时候调用.
  • 原型模式: 通过clone而非new来出啊关键object
  • 对象池模式： 核心是对象服用，不用时暂存留着后续使用。
