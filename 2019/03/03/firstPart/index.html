软件构造过程中的多维度视图

多维度软件视图

按阶段划分： build and run-time vies (构造时/运行时视图)
按动态性划分： moment and eriod views （时刻/阶段视图）
按构造对象划分： code and component views （代码/构件视图）

需要关注的是每个视图的元素、关系和模型

build-time

代码级视图：源代码 源代码如何通过基本程序块（如函数，类，方法，接口等）进行逻辑组织，以及它们之间依赖关系代码的逻辑组织
组件级视图：体系结构–源代码如何按文件、目录、包、库以及它们之间的一拉关系进行物理组织代码的物理组织
时刻视图：源代码和组件在特定时间内的样子特定时刻的软件形态
期间视图：它们如何随着时间的推移而发展/变化。软件形态随时间的变化

Build-time,moment and code-level view的结合

即从源代码的角度，从其基本的程序模块（包括函数，类，方法，等等）以及他们之间的联系，看源代码的逻辑组织。
具有三个内在联系的形式：词汇层面，语法层面，语义层面。
词汇层面： 使用半结构化的代码：近乎自然语言的风格+遵循特定的编程语法。
语法层面：常用抽象语法树（AST），来表示一个半结构化的源代码。
语义层面：考虑源代码要实现的目标，将源代码与现实世界联系起来，使用图形化或形式化的方法，来表达”需求“与”设计“思想，再将其转化为code。主要为面向对象的分析与面向需求的设计。
用于表达“需求”和设计思想，再转化为code

build-time,period and code-level view

随着时间观察变化
例如code churn(代码变化)

build-time,moment and component view

从代码的物理结构（源代码、文件、包和库等)看代码的物理组织

  • 源代码再物理结构上一文件的形式组织，更上层是以目录形式组织
  • 文件再逻辑上被封装为Package，compones和子系统
  • 可服用的模块以库的形式存储
    在执行build-time的时候需要执行静态链接，在执行时无需提供库文件，动态链接时在run-time时执行的

库

库存储在自己的磁盘文件中，持有一组可以被其他程序调用的代码函数
开发人员并不总是构建单个的可执行程序文件，而是将自定义开发的软件和预构建的库放到单个程序中去

库的来源
  • 操作系统提供
  • 编程语言提供
  • 第三方公司
  • 自己编写

Maven repositor一个JAVA的开源项目

build-time,period and component-level view

这一方面主要关注的是软件实体随时间变化的趋势，主要体现在版本和SCI上

Runtime Views

运行时即程序被载入目标机器开始运行的时候
代码层面：即逻辑实体如何在内存中呈现
构件层面：即物理实体在物理硬件环境重如何呈现
时刻层面：逻辑/物理实体在内存/环境中特定时刻的形态变化
阶段层面：逻辑/物理实体在内存/环境中特定随时间的形态变化

从run-time角度看软件的一些高级概念

  • 可执行程序： CPU执行的机器可读指令序列以及相关的数据值
    • 原生机器码
  • 库： 可以由不同程序使用的常用对象的集合
  • 配置和数据文件：提供可以从程序可以加载的有用数据和配置信息，是不可执行文件
  • 分布式程序： 这种类型的软件可由多个可执行程序组成，这些程序可以通过网络相互通信，或者只是在同一台机器上运行多个进程
